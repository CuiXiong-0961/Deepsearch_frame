# tools

## 定位
负责外部工具的统一接入、调用治理与执行安全控制。

## 负责内容
- 统一注册与调用工具能力（如 web_search、pdf_parser、python_executor、text2sql）。
- 处理工具参数校验、输入输出标准化与错误映射。
- 提供执行审计信息（调用日志、耗时、失败原因、重试记录）。
- 对高风险能力进行隔离与权限控制，保障运行安全。

## 总体实现思路
- 采用工具目录与统一调用门面，屏蔽不同工具的协议差异。
- 为每个工具定义清晰契约：输入字段、输出字段、错误语义。
- 在调用层增加稳健性策略（超时、重试、降级、熔断）提升可用性。
- 将结果转化为结构化记录，便于 Memory 入库与后续回溯。

## 边界约束
- 不决定业务流程分支，不做规划与反思判断。
- 只提供“可控、可追踪、可复用”的工具执行能力。
