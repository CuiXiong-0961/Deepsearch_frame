# orchestrator

## 定位
负责全链路流程调度与状态机编排，是系统执行入口与主循环控制层。

## 负责内容
- 初始化任务上下文并启动执行流。
- 调用 Planner 生成初始 Plan。
- 按 subtask 组织循环：检索 -> 反思 -> 需要时再检索 -> 产出小结。
- 汇总所有 subtask 小结并触发 Synthesizer 生成最终报告。
- 维护运行中的全局状态（任务、计划、当前子任务、文档池、小结、迭代次数等）。
- 统一处理异常、重试、超时、迭代上限与提前终止条件。

## 总体实现思路
- 采用图式编排（如状态机/工作流图）组织各模块节点，节点之间仅通过共享状态通信。
- 每个节点职责单一：只完成本步骤逻辑并读写必要状态，不直接耦合其他节点内部实现。
- 在循环路径上设置清晰分支条件（继续检索/进入总结/终止），防止无界循环。
- 为关键状态迁移记录事件，支持执行回放、诊断与可观测性。

## 边界约束
- 不承担具体检索实现、不承担具体工具执行细节、不负责报告写作细节。
- 只负责“何时调用谁、输入输出如何衔接、流程何时结束”。
